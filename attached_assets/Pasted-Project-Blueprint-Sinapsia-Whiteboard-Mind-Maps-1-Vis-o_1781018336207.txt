Project Blueprint: Sinapsia (Whiteboard & Mind Maps)1. Visão Geral do ProdutoO Sinapsia é uma ferramenta de mapas mentais e diagramação em tela infinita com fricção zero. O usuário deve ser capaz de criar, desenhar (com foco em touch/tablets) e compartilhar links sem a necessidade de criar contas ou fazer login.2. Stack Tecnológica DefinidaFrontend & Roteamento: Next.js 14 (com App Router) + Tailwind CSS.Motor do Canvas: Tldraw (biblioteca React que entrega canvas infinito, dark mode, ferramentas de desenho touch, texto e formas nativas).Banco de Dados & Tempo Real: Firebase Realtime Database (Plano Spark). Usado para salvar o estado do canvas e sincronizar cursores/traços instantaneamente entre múltiplos usuários no mesmo link.Armazenamento de Mídia: Cloudinary (Cloud Name: di3lqsxxc). Usado via Unsigned Upload Presets para que o frontend envie imagens/GIFs diretamente para a nuvem sem passar por rotas de backend.3. Arquitetura de Rotas (Next.js App Router)RotaDescriçãoComponentes Principaisapp/page.tsxLanding page simples. Contém apenas um botão "Criar Novo Mapa" que gera um UUID aleatório e redireciona o usuário.Botão de ação com redirect.app/b/[id]/page.tsxRota dinâmica do mapa. Lê o id da URL para puxar os dados do Firebase. Verifica se o parâmetro ?mode=view está presente para travar o mapa como "Apenas Leitura".Componente <Canvas boardId={id} readOnly={...} />4. Estrutura de Dados e Fluxo de MídiaA. Nó do Firebase Realtime DatabaseO estado de cada mapa será salvo em um nó plano para máxima performance de sincronização:JSON"boards": {
  "UUID_DO_MAPA": {
    "document_state": {},  // Objeto JSON puro gerado pelo Tldraw
    "updated_at": 1717945200000
  }
}
Otimização: O envio de dados do Canvas para o Firebase deve usar uma função de debounce (ex: 300ms) para não estourar o limite de conexões por milissegundo enquanto o usuário arrasta elementos.B. Lógica Híbrida de Imagens e GIFsPor Link (URL externa): Se o usuário colar um link direto da internet, o Tldraw renderiza o asset diretamente usando a URL informada.Por Upload (Arquivo local): Se o usuário arrastar um arquivo para a tela, o app intercepta o evento, faz um POST fetch direto para a API do Cloudinary usando o preset Unsigned, recebe a URL pública de volta e a injeta como um asset de imagem no Tldraw.5. Variáveis de Ambiente Necessárias (.env.local)O projeto precisa das seguintes chaves configuradas no ambiente do Replit:NEXT_PUBLIC_CLOUDINARY_CLOUD_NAME=di3lqsxxcNEXT_PUBLIC_CLOUDINARY_UPLOAD_PRESET (Preset Unsigned configurado no Cloudinary)Chaves de configuração padrão do Firebase (API_KEY, DATABASE_URL, PROJECT_ID, etc.)🤖 Comando para mandar no chat do Replit AI:"Baseado no blueprint acima, crie a estrutura inicial do projeto Next.js.
